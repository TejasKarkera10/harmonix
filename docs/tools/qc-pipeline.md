# QC Pipeline

**Stage:** 03 | **Mode:** SLURM parallel | **Time:** 2–3 hrs

---

## What it does

The most compute-intensive stage. Runs full signal quality checks across all subjects and channels simultaneously using SLURM array jobs on ERISTwo. Produces per-subject, per-epoch QC metrics for every signal domain, stratified by sleep stage.

---

## Usage

```bash
harmonix qc --sl <sample_list> [options]
```

**Example:**
```bash
harmonix qc --sl sl/s0.lst --cmap cmd/cmap.txt --out tmp/ --jobs 20
```

---

## Parameters

| Parameter | Required | Description |
|-----------|----------|-------------|
| `--sl` | Yes | Path to Luna sample list |
| `--cmap` | No | Path to channel alias file (default: `cmd/cmap.txt`) |
| `--out` | No | Output directory for `.db` files (default: `tmp/`) |
| `--jobs` | No | Number of SLURM array jobs (default: 20) |
| `--partition` | No | SLURM partition (default: `short`) |
| `--readme` | No | README path to auto-populate §4 (default: `README.md`) |

---

## Checks performed

### Signal-level checks (all channels)

| Check | Method | Soft flag | Hard flag |
|-------|--------|-----------|-----------|
| Flatline detection | Hjorth activity (SIGSTATS) | >20% epochs flat | >40% epochs flat |
| Amplitude range | STATS vs dataset IQR | >3× IQR | >10× IQR |
| Clipping | SIGSTATS clip detection | Any clipping | >10% epochs clipped |
| Sample rate | HEADERS vs expected | — | Unexpected SR |
| Unit labels | HEADERS | Mismatch warning | — |

### ECG-specific checks

| Check | Method | Soft flag | Hard flag |
|-------|--------|-----------|-----------|
| Polarity | HRV P_INV | P_INV 0.3–0.7 | — |
| R-peak quality | HRV NP_TOT | Low R-peak count | Zero R-peaks |
| HR plausibility | HRV HR | HR <40 or >100 | HR <20 or >150 |

### Staging checks

| Check | Method | Soft flag | Hard flag |
|-------|--------|-----------|-----------|
| EEG-stage consistency | SOAP kappa | kappa <0.4 | kappa <0.2 |
| Sleep efficiency | HYPNO | SE <50% or >99% | SE = 0% |
| Duration match | HEADERS vs staging | >10 min mismatch | >30 min mismatch |
| Stage coverage | HYPNO | Gaps in staging | No staging |

---

## Sleep stage stratification

QC metrics are computed both overall and stratified by sleep stage. This matters because:

- Wake epochs have expected movement artifact — flatline during wake is not a signal failure
- REM breathing is naturally irregular — strict morphology thresholds would false-positive
- NREM (N2/N3) is the most stable — best for assessing true signal quality

---

## SLURM job structure

The pipeline splits the sample list across N array jobs:

```bash
# auto-generated by harmonix qc
sbatch --array=1-20 cmd/run_qc.sh
```

Each job processes `ceil(N_subjects / N_jobs)` subjects and writes a `.db` file to `tmp/`. Results are merged after all jobs complete.

---

## Output

**`tmp/qc_*.db`** — per-job Luna output databases

**`res/qc_summary.txt`** — compiled per-subject QC metrics

**`res/review_queue.txt`** — flagged subjects appended with category `[QC]`

**`README.md §4`** — populated with signal summary statistics and distribution plots
